    The renderer is a pure function of the contract: render(contract) → surface. AIO types map to canonical defaults
    (display-collection → list, display-value → value block, trigger-action → button,
    single-select → option group, text-entry → field). Data is the embedded Projector output for the scenario —
    not placeholder text. Transitions make it clickable. Coupling a §11 manifest would swap these defaults for real components
    without touching the contract.
